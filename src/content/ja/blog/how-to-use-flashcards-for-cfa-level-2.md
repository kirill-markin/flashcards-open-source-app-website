リーディングも読んだし、式の系統も分かっている。それでもCFA Level IIで落とすことがあります。理由はたいてい、vignetteのたった1文が前提を丸ごとひっくり返していたからです。年金調整が比率を動かす。valuationの前提が変わって、モデルに入れる入力が変わる。倫理の細部ひとつで、いちばんもっともらしい選択肢が誤答になる。

そこで2026年の **CFA Level II フラッシュカード** が効いてきます。ただし、Level Iの癖をそのまま持ち込んだデッキでは足りません。Level II向けの良いデッキは、item setの中にある分岐点を拾える必要があります。どの一文で比率の向きが変わるのか。どの前提でvaluationの結論が反転するのか。どのportfolio制約で、いちばん素直に見える答えが使えなくなるのか。そこに反応できる形で作る必要があります。

公式の [CFA Institute Level II exam page](https://www.cfainstitute.org/programs/cfa-program/candidate-resources/level-ii-exam) は、試験形式をかなり率直に説明しています。Level IIは **22 item sets**、**88 multiple-choice questions**、そして **132分のセッションが2回** で構成されます。この22 item setsのうち **20セットが採点対象**、**2セットはtrial item sets** です。トピック配置はランダムで、**誤答ペナルティはありません**。さらに、合格者の多くが各レベルに **300時間超** を使っていると報告されています。これはLevel Iとは違う記憶の課題です。曖昧な見覚えでは足りず、長いケースの中で正しく引き出す必要があります。

## まずはLevel IIの問題の形に合わせる

Level IとLevel IIのいちばん大きな違いは、単に「内容が難しい」ことではありません。問題の構造です。

Level Iなら、1つの事実、1つの式、1つの倫理上の区別に切り分けやすい場面が多くあります。Level IIでは、それらがitem setの中に包まれます。vignetteを読み、複数の情報をワーキングメモリに置いたまま、同じ設定から計算、解釈、会計上の影響、portfolio判断を問う問題に続けて答えます。

だから **CFA Level II向けフラッシュカード** の作り方も変えるべきです。

デッキはvignette全体を覚え直させる必要はありません。プレッシャーの中で崩れやすい部分だけを残せば十分です。

- どの細部が答えの方向を変えるのか
- 実際に問われているLOSは何か
- この文脈で正しい式はどれか
- その計算からどんな解釈が導かれるか
- どのdistractorが毎回もっともらしく見えるか

Level I版の流れを先に見たいなら、[2026年版 CFA Level I向けフラッシュカードの使い方](/ja/blog/how-to-use-flashcards-for-cfa-level-1/) がそのまま補助記事になります。Level Iのほうがまだ素直です。Level IIは、より解釈寄りで、よりケースベースで、曖昧な想起にずっと厳しくなります。

## 2026年の配点を復習予算として使う

公式の [2026 Level II exam page](https://www.cfainstitute.org/programs/cfa-program/candidate-resources/level-ii-exam) には最新のトピック配点が載っています。私はこれを、全リーディングをカード化する命令ではなく、復習予算として扱います。

| CFA Level II のトピック | 2026年の公式配点 | フラッシュカードで効きやすい使い方 |
| --- | ---: | --- |
| Quantitative Methods | 5-10% | 回帰の前提、時系列の解釈、モデル選択の区別 |
| Economics | 5-10% | 通貨の関係、マクロの伝達ロジック、シナリオごとの影響 |
| Financial Statement Analysis | 10-15% | 会計調整、比率への影響、年金や持分法まわりの細部 |
| Corporate Issuers | 5-10% | 資本構成の影響、ガバナンスの区別、投資判断への帰結 |
| Equity Investments | 10-15% | valuationの入力、justified multipleの考え方、residual incomeの解釈 |
| Fixed Income | 10-15% | スプレッド指標、イールドカーブ変化、信用分析、durationとconvexityの解釈 |
| Derivatives | 5-10% | forwardとoptionのペイオフ、ヘッジ設定、価格関係 |
| Alternative Investments | 5-10% | 不動産やprivate capitalの指標、手数料構造、戦略の違い |
| Portfolio Management | 10-15% | リスク予算、active returnの解釈、portfolio制約、パフォーマンス要因分析 |
| Ethics | 10-15% | シナリオの手がかり、義務の衝突、mosaic theoryの境界、開示判断 |

この表が重要なのは、Level IIの勉強時間が本当によく消えるからです。配点が高く、しかも記憶負荷が重いトピックは、復習キューでより多く出るべきです。配点が低くてもミスが続くなら、枚数を増やすのではなくカードを鋭くします。

私なら週に1回、トピック別のカード枚数もざっくり数えます。Fixed Incomeのカードが多すぎるのは、努力の証拠とは限りません。単にその分野のカード作りがしやすくて、デッキがそちらへ流れただけかもしれません。

## カードにはLOS、トピック、失敗パターンのタグを付ける

これはLevel IIでかなり効く改善です。

公式の [2026 Level II topic outline PDF](https://www.cfainstitute.org/sites/default/files/docs/programs/cfa-program/2026-l2-topics-combined.pdf) には、現行の reading と learning outcomes が載っています。この構造をそのまま使います。カードを単に `equity` や `fixed-income` で分類するのではなく、LOSまたは reading、トピック、そしてどんなミスだったかまでタグ付けします。

| タグ層 | 例 | 役立つ理由 |
| --- | --- | --- |
| トピック | `fsa`, `equity`, `ethics`, `portfolio` | 偏りや弱点をすぐ見つけられる |
| LOS または reading | `los-fsa-4`, `reading-22` | どのカリキュラム項目に戻るべきか分かる |
| カードの役割 | `formula`, `interpretation`, `vignette-cue`, `mock-miss` | 計算カードと判断カードを分けられる |
| 失敗パターン | `sign-error`, `wrong-multiple`, `missed-adjustment`, `ethics-trap` | 本当に壊れたポイントが見える |
| 出典 | `cfa-qbank`, `mock-1`, `eoc`, `blue-box` | どの教材由来のミスか追いやすい |

このタグ構成なら、管理不能になるほど増えず、しかも十分に具体的です。実際のLevel IIのミスの感触にもかなり合っています。問題はたいてい「Equityが苦手」ではありません。もっとこうです。

- justified multipleを毎回取り違える
- 年金の細部を読んでいるのに比率の影響を落とす
- スプレッド自体は合っているのに解釈を逆にする
- 復習後なら倫理の答えが分かるが、vignette中では拾えない

こういうもののほうが、章タイトルよりずっと良い想起ターゲットになります。

デッキ構成がすでに散らかっているなら、[2026年版 フラッシュカードの整理術](/ja/blog/how-to-organize-flashcards/) を先に読む価値があります。カード枚数が増え切る前に整えたほうが、あとでかなり楽です。

## vignetteの分岐点からカードを作る

Level IIのitem setは、巨大カードを作りたくさせます。

vignetteの半分を表に貼り、解説を裏に丸ごと貼って、「ケースベースだから細部も必要」と自分に言い聞かせる人は多いです。でも、それで起きる復習は想起ではなく、ただの再読です。

私なら、多くのitem setのミスを1つの分岐点まで削ります。

例:

- 表: residual income の vignette で、justified P/B を押し上げやすい前提は何か?
- 裏: residual income の持続性が高いこと、または required return に対して収益性が強いこと。

- 表: FSA の item set で、年金処理が変わったとき最初に確認すべきことは何か?
- 裏: どの財務諸表項目と比率が動くか、その変化が利益、資本、キャッシュフロー区分、またはその複数に影響するか。

- 表: fixed-income の spread 問題で、spread拡大が強気材料になるのはどんなときか?
- 裏: それ単体では強気とは言えない。required return、リスク補償、spread縮小後の relative value など、文脈が必要。

どれも小さいカードのままです。ただ、Level Iの定義丸暗記ではなく、Level IIの判断ロジックから作られています。

## 式カードは想起だけでなく解釈まで試す

多くの **CFA Level II フラッシュカード** が弱いのは、formula sheetと同じ理由です。式を再現できても、ケースの中でその結果をどう読むかが抜けています。

Level IIでは、1つの式から少なくとも3つの記憶タスクが生まれます。

- 式そのものを思い出す
- vignetteから正しい入力を選ぶ
- 出た結果が何を意味するかを解釈する

私はたいてい、これを別々のカードに分けます。

たとえば、1つのvaluationミスから次のように分けられます。

1. formulaカード
   表: fundamentals に基づく justified forward P/E の関係はどうなるか?
   裏: Gordon型の前提で、expected payout、required return、expected growth を結び付ける。

2. 入力選択カード
   表: justified multiple の vignette で、いちばん起きやすい入力ミスは何か?
   裏: ケースの前提から導くべきモデル入力ではなく、市場で観測された multiple や過去値を拾ってしまうこと。

3. 解釈カード
   表: required return が上がり、expected growth が変わらないとき、justified forward P/E はどうなるか?
   裏: 低下する。成長に対して分母側が広がるから。

厳密な式そのものは reading ごとに違いますが、型は同じです。Level IIで点を取りやすい人は、数字から意味へ移るたびに、頭の中で reading 全体を組み直したりしません。

## CFA Instituteの演習問題と模試を主な材料にする

公式の [CFA candidate resources page](https://www.cfainstitute.org/programs/cfa-program/candidate-resources) によると、2026年2月試験向けのLearning Ecosystemには、curriculum、practice questions、mock exams、study planner、flashcards、Practical Skills Modules が含まれています。材料は十分あります。

同時に、デッキ作りのルールもかなり明確です。カードの大半は、ハイライトした段落ではなく、自分が実際に解いた問題から作るべきです。公式のLearning Ecosystem flashcardsを使っているとしても、それは補助にとどめます。個人デッキの中心は、実際に点を落としたitem setと、その解釈ミスであるべきです。

良い **CFA 模試 復習** 用カードは、たいてい次のどこかから生まれます。

- CFA Instituteの演習問題で、再現性のある理由で落としたもの
- 章末問題で、解説を読んで長く残る区別が見えたもの
- 手法自体は合っていたのに、模試で解釈を外したもの
- 会計やportfolioの細部ひとつで結論が何度も反転したitem set

この材料の偏りは重要です。Level IIには、見たことがある気がする問題が多いからです。ミスは、「見覚え」が信頼できなくなった場所を示してくれます。

誤答からカードに落とす全体フローを見たいなら、[2026年版 演習問題をフラッシュカードに変える方法](/ja/blog/how-to-turn-practice-questions-into-flashcards/) が、その抽出作業をもっと詳しく扱っています。

## 1つのitem setのミスから3枚の良いカードが作れる

Level IIの1回のミスには、もっと小さい失敗がいくつも入っています。

たとえば、多国籍企業会計やresidual incomeのvignetteを落としたとして、実際に壊れたのは次のどれかもしれません。

- 会計処理を変える1行を見落とした
- valuationアプローチを選び間違えた
- 出た結果の方向を逆に読んだ

こういうときは、巨大なケース要約1枚ではなく、小さなカード2枚か3枚に分けます。

私なら、次の型を使います。

| カード種別 | 表の聞き方 | タグ例 |
| --- | --- | --- |
| Vignette cue | このタイプのケースで、最初に手法や答えを変える細部は何か? | `fsa`, `vignette-cue`, `mock-miss` |
| Formula setup | この状況では、どの式や入力を使うべきか? | `equity`, `formula`, `wrong-input` |
| Interpretation | 計算後、その結果は何を意味するか? | `portfolio`, `interpretation`, `sign-error` |
| Ethics judgment | 実際に有効になっている義務や開示の論点は何か? | `ethics`, `ethics-trap`, `cfa-qbank` |

このやり方なら、復習は最後まで回せる速度を保ちつつ、ちゃんと意味のある粒度になります。

## 疲労、時間切れ、単純な不注意のためのカードは作らない

Level IIのミスには、記憶のミスもあります。そうでないものもあります。

私なら、主な原因が次のどれかだった場合は、基本的にフラッシュカードを作りません。

- セッション後半で時間が足りなくなった
- vignetteを流し読みして、重要だと分かっていた単語を落とした
- そのreading自体をまだ学び切っていなかった
- 手法は理解していて、calculator入力を1回だけ間違えた
- 長い模試のあとで単純に頭が回っていなかった

もちろん、こうした問題も対処は必要です。ただし、対策は別です。

- timed item-set practice を増やす
- セッション間の休息を確保する
- calculatorの手順を整える
- 長いケースでは最初の読みを少し丁寧にする

フラッシュカードが効くのは、そのミスが次回もっと正確に引き出したいものを示しているときです。

- 手がかり
- 区別
- LOSレベルの概念
- 会計上の影響
- 式の解釈
- 倫理の境界

このフィルターがあると、デッキが「うんざりした勉強時間の保管庫」にならずに済みます。

## デッキは1つの混合型にして、崩れたトピックだけ後で絞る

Level II本番ではトピック配置がランダムなので、最初から大量のサブデッキに分ける必要はあまりありません。どうしてもその構造が必要だと分かっている場合を除けば、私はそうしません。

メインデッキ1つとタグで十分なことが多いです。

そのほうが、復習体験も本番に近づきます。ethicsのカードの次にderivativesの解釈カード、その次にfixed-incomeのspreadカード、その次にFSAの調整カードが来る。こういう混合想起には意味があります。

トピックが崩れ始めたら、そこで絞ります。

- `ethics + mock-miss`
- `portfolio + interpretation`
- `fsa + los-fsa-4`
- `fixed-income + sign-error`

このほうが、カードを毎回移動させるより柔軟です。

アプリの基本運用がまだ固まっていないなら、[スタートガイド](/ja/docs/getting-started/) が入り口になります。ここで重要なのは、同じ製品の中にカード編集、タグ、FSRS復習がそろっていることです。その概要は [機能ページ](/ja/features/) でも確認できます。

## Level IIでは、難しさが均一ではないからFSRSが合う

Level IIのデッキは、かなり早い段階で性質がばらつきます。

2回の復習で安定する簡単な定義もあります。3週間ずっと混ざり続けるvaluationの関係もあります。ethicsのカードも、単独なら簡単なのに、次のvignetteで client、employer、research pressure が一度に入ると急に崩れます。

だから **FSRS CFA 勉強法** はこの試験と相性が良いです。簡単なカードは遠ざけて、滑りやすいカードは近くに残したいからです。

まだ設定していないなら、次の2本がそのまま相方になります。

- [2026年版 FSRSで試験勉強する方法](/ja/blog/how-to-study-for-an-exam-with-fsrs/)
- [2026年版 FSRS設定の考え方](/ja/blog/fsrs-settings/)

重要なのはアルゴリズムの豆知識ではありません。デッキ運用の規律です。

- カードは少量ずつ追加する
- 曖昧なカードはすぐ削除するか書き直す
- 終盤に大きなかたまりを一気に取り込まない
- 期限が来た混合カードを毎日回す

Level II対策は、もともと動く部品が多いです。スケジューラはその運用を落ち着かせるために使うべきであって、コンテンツをため込む言い訳にするべきではありません。

## 毎週回せるLevel IIフラッシュカード運用

私なら、週次の流れはわざと地味に保ちます。

1. curriculum または予備校ノートの1つの reading、または小さめの学習ブロックを進める。
2. 前提がまだ新しいうちに、すぐCFA Instituteの演習問題を解く。
3. LOSレベルの区別、式の解釈ミス、繰り返し起きる誤答だけからカードを作る。
4. 各カードに、トピック、LOSまたは reading、失敗パターンのタグを付ける。
5. FSRSで期限が来たカードを毎日復習する。
6. 週末に、1つのトピックが静かにデッキを占領していないか、または同じ種類のミスが増えていないかを見る。

この最後の見直しは、見た目以上に重要です。トピックの配分やミスの偏りが変なら、早めに工程を直します。次の模試まで放置して、「弱いカードの半分が実は同じ会計調整を5通りに書き直しただけだった」と気付くのは遅いです。

毎日のキューがだんだん重くなってきたら、[2026年版 フラッシュカードを速く復習する方法](/ja/blog/how-to-review-flashcards-faster/) が、カードを増やす前の整理として役立ちます。

## Flashcards Open Source App がこの流れに合う理由

[Flashcards Open Source App](/ja/) は、この運用と相性が良いです。Level II受験者が必要としがちな部品を、すでに1か所で持っているからです。

- 小さく保てる表裏カード
- topic、LOS、mock-miss を追えるタグ
- 難易度のばらつきに合うFSRSスケジューリング
- すぐ始めたい人向けの公開ホスト版アプリ
- 学習環境を自分で持ちたい人向けのオープンソースのコードベースとセルフホスティングの道筋
- プロジェクト全体の offline-first clients。カードを作った机の前以外でも復習しやすいです

この製品を使っても、Level IIそのものが簡単になるわけではありません。約束できるのはそこではありません。役に立つのは、問題復習、カード修正、間隔反復の流れを、必要以上にたくさんのツールへ分散させずに済むことです。始めるなら、まずはデッキ1つ、タグ層3つ、そして直近10件の演習または模試のうち、実際に想起の弱さが出たミスから始めるのがきれいです。

## デッキは判断に向けて尖らせる

試験直前でも役に立つ **CFA Level II フラッシュカード** を作りたいなら、試験が何度も迫ってくる判断に合わせて作るべきです。

- どの手がかりが手法を変えるのか
- 実際に動いているLOSは何か
- 式に入るべき入力はどれか
- その出力が本当は何を意味するのか
- どの模試のミスが繰り返しているのか

私が信頼するのは、そういう形です。

小さいカード。明確なタグ。混合復習。実際に解いたitem setから作ったデッキ。

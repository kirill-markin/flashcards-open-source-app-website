把 Anki 和 Quizlet 并排打开，你通常还没点进去，就已经能猜到它们之间的取舍。一个看起来功能强，但也有点倔。另一个更顺滑，更像面向大众消费者的产品。

这种分裂已经持续很多年了。仔细想想其实很奇怪。Flashcards 并不是什么不可能做好的产品类别。它本质上就是卡片、复习队列、排程逻辑，再加上一套习惯循环。按理说，我们早该有更多靠谱选项了。

但现实是，大多数人到现在还是只能在三条路里选一条：

- 如果想要严肃的间隔重复，并且能忍受老派 UX，就选 Anki
- 如果想要更干净、更主流的产品体验，并且接受封闭平台，就选 Quizlet
- 如果想要所有权、自托管，以及一个真正像这个时代软件的产品，就选较新的开源 flashcards app

第三类，终于开始变得有意思了。

## Anki vs Quizlet，本质上是在权衡“强大功能”和“产品体验”

如果你搜索 **Anki vs Quizlet**，大多数比较文章都会试图把差异压缩成一张功能清单。

这其实没说到重点。

真正的区别，在技术之前，先是感受上的区别。

Anki 给人的感觉，是由更关心学习系统本身、而不是第一印象的人做出来的工具。Quizlet 则更像是一个为了更容易上手、更容易分享、也更容易卖给更广泛人群而打造的产品。

这两种选择都说得通。也都要付出代价。

## 为什么很多人至今仍然选择 Anki

对于认真对待记忆训练的人来说，Anki 依旧是默认答案。

语言学习者在用它。医学生在用它。准备高难考试的人也在用它。不是因为它好看，而是因为它确实有效，并且已经在多年使用中经受了验证。

如果你想要最成熟、口碑最稳的 **间隔重复应用**，Anki 依旧值得尊重。

它现在仍然做得好的地方有：

- 复习纪律性很强
- 在间隔重复领域口碑扎实
- 已有庞大的社区
- 有很多现成卡组、插件和其他用户的经验
- 更像本地优先工具，而不是“所有东西都属于平台”

它的弱点不是算法，而是产品体验。

你当然可以习惯 Anki。很多人也确实习惯了。但在 2026 年，“用久了就习惯了”并不能算一句特别有说服力的产品好评。

## 为什么很多人至今仍然选择 Quizlet

Quizlet 的优势在于易用性。

它更轻、更简单，也更接近大多数人对现代学习应用的预期。如果你只是想快速做几组卡片、分享出去、几乎不用配置就开始学习，那 Quizlet 的吸引力非常明显。

这也是为什么很多人几乎一上来就对 Anki 失去耐心，却会继续留在 Quizlet。

但它的代价也正如你对一个封闭主流产品所预期的那样：

- 更少的所有权
- 更少的灵活性
- 产品方向由别人决定
- 你的学习系统活在别人的商业模式里

如果只是轻度学习，这也许是完全可以接受的交易。

但如果是长期的个人知识积累，我觉得这笔交易就没那么划算了。

## Anki 和 Quizlet 都开始显得不够用的地方

这通常就是人们开始寻找 **Anki 替代品** 或 **Quizlet 替代品** 的起点。

有些人想要 Anki 那种严肃学习能力，但不要那么过时的感觉。

有些人想要 Quizlet 的易用性，但不想接受封闭平台的代价。

还有些人想要一样这两个产品都处理得不够好的东西：真正的所有权。

Flashcards 不是一次性内容。几年下来，它们会慢慢变成一张地图，记录你在学什么、什么总是会忘、哪些主题值得反复回来、以及你的理解如何随着时间变化。

这是有价值的数据。我不太喜欢把这种东西当成“租来的平台内容”。

## 现代开源 flashcards app 改变了什么

[Flashcards](https://flashcards-open-source-app.com/) 代表的是我希望这个类别里更常见的一种方向：一个现代化的 **开源 flashcards app**，带有间隔重复、明确记录的自托管路径，以及一个足够简洁、不至于发展成博物馆的软件表面。

现在公开可用的版本，对自身阶段是很诚实的。它还早期，但已经不只是一个只能在浏览器里演示的原型。

你今天已经能清楚拿到这些能力：

- 在托管 web app 里创建正反面卡片
- 用 FSRS 排程复习到期卡片
- 基于 workspace 数据和文件附件使用 AI chat
- 通过无密码 email OTP 登录
- 通过 discovery-first API flow 连接终端 agent
- 从主仓库运行支持离线优先同步的 iOS 客户端
- 如果你想掌控全局，可以自托管整套栈
- 使用面向 workspace 初始化和 SQL 访问的精简外部 agent API

这些已经足够覆盖真正重要的核心循环：创建卡片、复习到期内容、明天继续。

更重要的是，它的整体方向也更符合软件现在应该走的路：离线优先客户端、开放架构，以及一个你可以真正检查、而不只是租用的软件系统。

## 自托管 flashcards 不只是业余运维爱好者的玩具

人们搜索 **self hosted flashcards** 时，通常并不是因为他们想给周末增加更多杂务。

他们通常是想要下面三件事之一：

- 掌控自己的学习数据
- 未来能在托管版和自托管版之间切换
- 确认这个产品不是一个黑箱

这是很合理的直觉。

开源 flashcards app 的好处在于，就算你一开始使用的是托管版，你也不会被困在那里。你可以查看代码、阅读文档、理解 API，并且始终保留选择权。

这一点在学习工具里比很多人承认的重要得多。学习系统往往会一用就是很多年。

## 2026 年，哪个才是最好的间隔重复应用？

这取决于你真正要解决的是什么问题。

如果你现在想要的是最成熟、最经验证的重度记忆工具，Anki 依旧很难被打败。

如果你想要最简单的主流体验，而且不太在意锁定效应，Quizlet 依旧是更直接的答案。

如果你想要一个更新的 **开源 flashcards app**，它更接近现代产品预期，有真实的自托管路径，并且把所有权保留在桌面上，那 Flashcards 会是更有意思的选择。

这并不是说新选择已经在所有维度上全面超越老牌产品。它还没有。

我的意思是，这个品类已经被同一种取舍困住太久了，而这条新方向是近几年第一次让我觉得在结构上更合理的答案。

## 一个实用对比：Anki、Quizlet 和 Flashcards

| 工具 | 最适合谁 | 主要优势 | 主要弱点 |
|---|---|---|---|
| Anki | 最看重复习质量的严肃学习者 | 已验证的间隔重复文化和深度 | 产品显得老旧 |
| Quizlet | 轻度或主流学习场景 | 更容易上手，消费者 UX 更顺滑 | 封闭平台，所有权更弱 |
| Flashcards | 想要现代开源软件和自托管选项的人 | 所有权、透明架构、聚焦的产品方向 | 仍是更早期的产品 |

这张表已经是比较诚实的版本了。

很多比较文章都会回避一件不太舒服、但很真实的事：有时候最好的选择，不是功能列表最长的那个，而是你真的能带着它生活很多年的那个取舍。

## 谁适合用哪个

如果你符合下面情况，就选 **Anki**：

- 你想要最成熟、最经验证的间隔重复流程
- 你不介意更老的 UX
- 你优先考虑深度和生态，而不是产品打磨

如果你符合下面情况，就选 **Quizlet**：

- 你想要更快上手的产品
- 你更在意便利，而不是所有权
- 你的学习流程更轻，不是那么系统驱动

如果你符合下面情况，就选 **Flashcards**：

- 你想要一个开源的 **Anki 替代品**
- 你想要一个不被平台完全锁死的 **Quizlet 替代品**
- 你现在或以后会在意自托管
- 你接受它还处于早期，但方向是对的

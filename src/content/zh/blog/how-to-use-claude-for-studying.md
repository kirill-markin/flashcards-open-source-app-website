上周 Claude 用大概六分钟帮我理顺了一道统计学题。第二天早上我重新拿纸推一遍时，还是在同一个地方卡住了。这个提醒倒是很有用。

大家一直在搜 **如何用 Claude 学习**，因为现在真正的瓶颈，早就不是“有没有人给你解释清楚”了。Claude 在讲解这件事上确实不错。更难的是，聊天窗口一关，你到底还能留下多少东西。

到了 2026 年，我真正信得过的流程其实很简单，也足够重复使用：先用 Claude 讲解、追问、暴露薄弱点，再把真正值得保留的部分放进一套像样的闪卡系统里，用牌组、标签和 FSRS 去复习。很多原本看起来不错的 AI 学习流程，往往就是在后半段悄悄散掉的。

## 2025 年 4 月 2 日之后，Claude 更适合拿来学习了

Anthropic 在 **2025 年 4 月 2 日** 推出了 **Claude for Education**，并把 **Learning Mode** 描述成一种导师式体验：它会带着你一步步推理，而不是一上来就把答案端出来。Anthropic 还提到 Learning Mode 可以在 Projects 里使用。这点很重要，因为 Projects 会给一整次学习过程一个稳定的容器，而不是把内容散落在一堆临时聊天里。

Projects 这一层现在也更实用了。Anthropic 的 Projects 帮助页面在 **2026 年 3 月 16 日** 更新后写得很明确：Projects 对所有 Claude 用户开放，免费账户也能用，只是免费用户最多创建五个项目。

这让今天的 **Claude 学习** 比一年前更像一套完整方法：

- 一个 Project 可以专门承载一门课、一场考试，或者一个主题
- Claude 可以直接基于你上传的材料来工作，而不只是依赖你仓促写下的摘要
- 在支持 Learning Mode 的情况下，它会更愿意逼你自己推理，而不是让你被动看答案

有个限制要说清楚：Learning Mode 并不等于 Claude 在任何场景下都会这样工作。如果你的套餐或学校没有开放这个模式，你还是可以在 Project 里用提示词模仿出类似的导师节奏。只是别把“提示词复刻”说成和产品功能本身完全一样。

不过，就算不夸大任何东西，这里也已经足够搭出一套扎实的 **Claude 学习流程**。前提是，你别把那段学习聊天本身误当成记忆系统。

## 每门课、每场考试或每个主题，单独建一个 Project

这是我最想保留的第一个习惯，因为它几乎会把后面每一步都变顺。

不要把所有科目全丢进一个巨大的 Claude 工作区里。最好是一门课一个 Project，一场考试一个 Project，或者一个边界很清楚的主题一个 Project。这样上下文更干净，后续追问更准，等你回头复习时也不会乱成一团。

如果是拿 **Claude Projects 学习**，我只会上传那些能让 Claude 更会提问，或者更能按正确深度讲解的材料：

- 你自己的课堂笔记
- 课程大纲或考试提纲
- 一份简短的学习指南
- 订正过的练习题
- 一次只放一章内容、一套幻灯片，或者一小段阅读材料

我不会因为上传框摆在那里，就在第一天把整个学期的东西全倒进去。

理由听起来有点无聊，但很重要。一个学习 Project 的任务，是帮 Claude 理解你此刻到底在学什么，不是把你从一月到现在碰过的每一份 PDF 都堆进去当仓库。

## 把 Claude 当导师用，不要只把它当摘要机器

**用 Claude 学习** 这件事，一旦做对，体验会很快好很多。

如果你能用 **Claude Learning Mode**，就直接用。Anthropic 把它定位成导师式工作流是有原因的。最有价值的时候，往往不是 Claude 给出那段润色过的完整答案，而是它先让你解释、比较、预测，或者自己先走完下一步。

如果你暂时没有 Learning Mode，也还是能在 Project 里用一句普通提示词，拿到大部分相同的行为：

> 像导师一样教我。一次只问一个问题。不要太早把完整答案告诉我。如果我答错了，明确指出我错在哪里，并且先让我再试一次，再完整解释。

这个小变化会把整场会话带到更有用的方向。Claude 不再只是一个讲得很快的解释器，而会更像一个陪你练习的人。

我不想让 Claude 一开场就给我一份整整齐齐的总结。我更想让它暴露出这些东西：

- 哪些内容我根本回忆不出来
- 哪些概念我会和相近内容混在一起
- 哪些点我只能“认出来”，却说不出来
- 哪些东西只有 Claude 讲完以后，我才觉得“哦，这个我本来会”

这才是 **Claude Learning Mode** 真正有用的地方。价值不在于对话有多舒服，而在于它给了你一份证据，告诉你理解到底是在哪里断掉的。

## 聊天记录本身不是目标

很多 **Claude 笔记转闪卡** 的流程，就是在这里开始走偏的。

人们做完一轮挺有帮助的会话，就把整段对话全复制出来，再让另一个工具把所有内容都做成卡。最后得到的通常是一副充满填充物的牌组：

- 热身用的解释
- 重复出现的提示
- 半成品答案
- 当时有帮助、以后却不值得留的例子
- 看着很完整、复习起来却很糟的总结段落

我更愿意把 Claude 会话当成“挖薄弱点”的过程，而不是导出整段聊天记录。

一次会话结束时，我真正想拿走的，是这样一份短名单：

- 漏掉的定义
- 搞混的比较点
- 我没法自己重新搭出来的公式结构
- 被我跳过去的步骤
- 我连续掉进去两次的陷阱

这才是那些以后仍然值得复习的 **Claude Learning Mode 闪卡** 的原材料。

## 一张卡只盯一个薄弱点，还是最有效

工具确实变了。写卡的基本规则几乎没怎么变。

一张来自 Claude 的好卡，通常只是老老实实地把一件事做好：

- 正面只有一个清楚提示
- 背面只有一个直接答案
- 带上足够的上下文，让它以后单独看也成立
- 不依赖你重新把整段聊天又读一遍

所以我不会把一次 Claude 导师会话硬做成二十张野心很大的迷你课程卡。我宁愿只留六张，每张只对准一个真实暴露出来的失误。

糟糕的版本通常会长这样：

- 解释这个主题。
- 总结这一章。
- 这里的关键思想是什么。
- 把整套方法完整讲一遍。

这些更像笔记提示，不像复习提示。

## 这是我真的会重复用的流程

最好的 **如何用 Claude 学习** 流程，应该短到新鲜感退掉以后你还愿意继续用。

我自己会用这个版本：

1. 为一门课、一场考试，或者一个主题创建一个 Claude Project。
2. 只上传和这个主题直接相关的笔记、幻灯片、阅读材料或订正后的题目。
3. 让 Claude 用导师模式来教，能用 Learning Mode 就优先用。
4. 在回答过程中，顺手记一个很小的薄弱点清单。
5. 会话结束后，让 Claude 只把这些薄弱点整理成普通的正反面候选卡。
6. 任何模糊的卡，立刻删掉或重写。
7. 把剩下的卡放进你的 flashcards app，再用 FSRS 复习。

第四步比很多人想得更重要。如果你不在当下记下那些答错和卡住的地方，整场聊天会显得很有效率，但你最后还是会忘记自己真正是被什么绊住的。

我会这样要求它生成候选卡：

> 只把这些薄弱点整理成闪卡。一张卡一个概念。正面要短。背面要直接。不要填充内容。不要编造事实。凡是仍然需要一整段话才能答完的，都跳过。

这会比把整段会话直接扔进去，再说一句“从这里帮我做闪卡”，效果好得多。

## Flashcards 是记忆保留层，不是什么假的 Claude 集成

这里得说得诚实一点，因为很多 AI 学习内容一写到这儿就开始打滑。

这里并没有什么特殊的 Claude-to-Flashcards 按钮，我也不会假装有。真正的交接动作很简单：把有用的薄弱点复制出来，清理干净，然后把最终卡片放进一个为复习而不是为对话设计的系统里。

这正是 [Flashcards](/zh/) 适合接手的地方：

- 正反面卡片的创建和编辑
- 用牌组和标签按课程、考试或薄弱点类型来组织
- 用 FSRS 做长期复习排程
- 需要时可用带文件附件的 AI chat 来清理粗糙笔记或卡片草稿
- 如果你想尽快开始，也有托管版 web app

如果你想先看产品概览，[功能页](/zh/features/) 是最快的入口。如果你准备直接开始用托管版应用，[入门指南](/zh/docs/getting-started/) 会把当前路径写清楚。

真正有用的是这套分工：

- Claude 负责讲解、追问，并暴露失误
- Flashcards 负责存储、组织、编辑，并安排那些值得留下的内容

这比期待一条聊天线程同时担任导师、笔记本、闪卡牌组和复习排程器，要耐用得多。

## 什么该上传给 Claude，什么该留给 Flashcards

我会把 Claude 用在“来源材料很多”的工作上，把 Flashcards 用在“需要长期记住”的工作上。

Claude 很适合这些事：

- 把一段难懂的解释讲简单
- 比较两个很像的概念
- 把凌乱笔记整理得更清楚
- 基于你上传的材料生成几道后续问题
- 找出哪些错误一直在重复

Flashcards 更适合这些事：

- 让最终卡片措辞保持稳定
- 把卡片放进牌组和标签里
- 直接复习到期内容，而不是反复打开同一段聊天
- 让 FSRS 决定下一次什么时候该回来

这样的分工能避免整套流程变得太戏剧化。你不会再把一次体验不错的 AI 对话，误当成真正的记忆系统。

如果你的主要来源就是你自己的笔记，可以继续看 [2026 年如何把笔记变成 Flashcards](/zh/blog/turn-notes-into-flashcards/)。如果你想看比 Claude 更宽一点的 AI 学习全貌，那篇 [2026 年如何用 AI 学习](/zh/blog/how-to-use-ai-to-study/) 会更合适。

## 最常见的错误，就是保存得太多

Claude 很容易让人产生一种“今天效率很高”的感觉。

你上传笔记。Claude 解释这一章。它问了几个还不错的问题。它重写了你的提纲。它又给了你一段很顺的总结。接着诱惑就来了：全都留着。

然后积压就从这里开始。

我自己会守的一条规则很简单：

- 保存那些答错的地方
- 保存那些回答明显变慢的地方
- 保存那些重复出现的混淆点
- 不要保存每一句当下看起来有用的话

这很重要，因为 FSRS 的确很擅长安排值得复习的卡。但它并不会让普通卡片突然变得毫无成本。

如果你已经知道牌组开始变太大了，那接下来读 [2026 年每天该新增多少张 Flashcards？](/zh/blog/how-many-new-flashcards-per-day/) 和 [2026 年落下 Flashcards 之后怎么追上进度](/zh/blog/how-to-catch-up-on-flashcards-after-falling-behind/) 会很顺。

## Claude 很擅长帮你想明白。Flashcards 更擅长帮你记住。

这是我目前能想到的最清楚的总结方式。

到了 2026 年，Claude 当然可以帮你学习。它可以换个角度解释概念，可以基于你上传的材料工作，可以比一页静态笔记问出更好的问题，也能把你理解还发软的地方直接照出来。

但它自己并不会替你搭起一套平稳的长期复习循环。

所以我理解的 **Claude 学习**，不是“用聊天取代闪卡”，而是“用聊天找出哪些东西值得做成闪卡”。

只要这条边界清楚，整套流程就能站得住：

1. 先用 Claude 把内容学明白。
2. 只留下真正暴露出来的薄弱点。
3. 把这些点做成小而清楚的卡片。
4. 后面的复习节奏交给 FSRS。

这是我真正会相信它下周仍然有效的 **如何用 Claude 学习** 方法，不只是今晚看起来很有帮助而已。
